BrowserSync

Transport

Point the URL at a JSON file in a folder you can write to. The folder is created if missing.

What to sync

Firefox keeps original visit dates; Chromium stamps imported visits at sync time (a browser limitation). History is only ever added, never removed.

When to sync

Chrome rounds intervals under 1 minute up to ~1 minute; Firefox allows shorter. For near-instant updates use the option below instead of a tiny interval.

What sync may change in this browser

Uncheck "Remove" if you never want sync to delete your local bookmarks.

Encryption

The transport (WebDAV/cloud/agent) only ever sees ciphertext. Enter the SAME passphrase on every device. ⚠️ If you lose it, the data is unrecoverable — keep an unencrypted export somewhere safe.

Filters (privacy)

Matches the domain and its subdomains. Already-synced entries on other devices aren't removed.

Safety
Other devices' open tabs

Enable "Open tabs" and sync to see tabs from your other devices here.

Backups

Backups are taken automatically before a sync removes bookmarks.

Sync inspector
Export / Import (offline migration)

Carry your data to a fresh browser with no server: export a file here, then import it on the new machine. Import only ever adds.

Device

"Receive only" mirrors the shared data here without sharing this device's; "Send only" is a master that uploads but ignores others.
